# הגדרת מדידה ודוחות GA4 למחשבון הצרכני

המדריך מיועד לנכס GA4 שמזהה המדידה שלו הוא `G-XF5X2QLB8L`. הוא מתאר את ההגדרה בממשק Google Analytics בעברית. שמות תפריטים עשויים להשתנות מעט בעדכוני ממשק.

## לפני שמתחילים

- אין ליצור ממדים כספיים או ממד לתוכן הודעת WhatsApp.
- אין ליצור ממד כפול אם פרמטר כבר רשום כממד מותאם אישית.
- כל הממדים המפורטים כאן הם בהיקף **אירוע**.
- לאחר יצירת ממד עשויות לחלוף 24–48 שעות עד שיופיע בדוחות.
- הנתונים אינם בהכרח רטרואקטיביים לתקופה שלפני יצירת הממד.
- קוד מפנה, כגון `roynetanel`, הוא מזהה שיווקי פנימי ולא מידע על לקוח.

## ממדים מותאמים אישית

היכנסו אל: **ניהול → הצגת נתונים → הגדרות מותאמות אישית → ממדים מותאמים אישית → יצירת ממד מותאם אישית**.

צרו את הממדים הבאים. בכל שורה הגדירו את ההיקף כ־**אירוע** ואת Event parameter בדיוק כפי שהוא מופיע בטבלה:

| שם תצוגה בעברית | Event parameter |
| --- | --- |
| מקור שיווק פנימי | `source` |
| סוג תנועה פנימי | `medium` |
| קמפיין פנימי | `campaign` |
| תוכן שיווקי | `content` |
| מילת חיפוש פנימית | `term` |
| קוד מפנה | `referrer_code` |
| מצב הקרן | `fund_status` |
| שיטת הפקדה | `deposit_method` |
| מטרות המשתמש | `goals` |
| מצב התוצאה | `result_status` |
| מיקום כפתור WhatsApp | `button_location` |

אין ליצור ממדים עבור הכנסה, הפקדה, צבירה, סכום מומלץ, הטבת מס, מספר טלפון או תוכן ההודעה.

## אירועים מרכזיים

היכנסו אל **ניהול → הצגת נתונים → אירועים** וסמנו כאירועים מרכזיים:

- `calculator_completed` — השלמת בדיקה.
- `whatsapp_clicked` — ליד פוטנציאלי והמדד העסקי החשוב ביותר כרגע.

אין לסמן את `landing_view` או `calculator_started` כאירועים מרכזיים. לחיצה על WhatsApp אינה הוכחה שההודעה נשלחה בפועל.

## משפך בדיקת קרן השתלמות

1. היכנסו אל **סקירה → ניתוחים → ניתוח משפך**.
2. תנו לניתוח את השם **משפך בדיקת קרן השתלמות**.
3. הגדירו משפך סגור וחלון זמן של 30 ימים.
4. הוסיפו לפי הסדר את השלבים `landing_view`, `calculator_started`, `calculator_completed`, `whatsapp_clicked` לפי התנאי "שם האירוע שווה".
5. הפעילו הצגת זמן מעבר בין שלבים.
6. כאשר הממשק מאפשר, הוסיפו פירוט לפי מקור שיווק פנימי, קמפיין פנימי או קוד מפנה.

המדדים לחישוב:

- שיעור התחלה = `calculator_started / landing_view`.
- שיעור השלמה = `calculator_completed / calculator_started`.
- שיעור מעבר ל־WhatsApp = `whatsapp_clicked / calculator_completed`.
- המרה כוללת לליד = `whatsapp_clicked / landing_view`.

## ביצועי קישורים וקמפיינים

1. צרו **ניתוח → טופס חופשי** בשם **ביצועי קישורים וקמפיינים**.
2. יבאו את הממדים: מקור שיווק פנימי, סוג תנועה פנימי, קמפיין פנימי, תוכן שיווקי, קוד מפנה ושם האירוע.
3. כאשר זמינים, הוסיפו גם מקור תנועה של סשן, אמצעי הגעה של סשן וקמפיין של סשן.
4. יבאו את המדדים: משתמשים פעילים, סשנים, מספר אירועים, אירועים מרכזיים ושיעור אירועים מרכזיים בסשן.
5. הגדירו כשורות: מקור שיווק פנימי, קמפיין פנימי, תוכן שיווקי וקוד מפנה.
6. הגדירו כערכים: משתמשים פעילים, סשנים, מספר אירועים ואירועים מרכזיים.
7. בגרסה הכללית הוסיפו מסנן "שם האירוע תואם לביטוי רגולרי" עם `^(landing_view|calculator_completed|whatsapp_clicked)$`.
8. שכפלו את הכרטיסייה ובגרסת הלידים סננו ל־`whatsapp_clicked` בלבד. גרסה זו מזהה אילו קישורים יצרו לידים פוטנציאליים.

## DebugView

הפעילו מצב Debug רק בדפדפן הבדיקה או באמצעות Tag Assistant; אין להפעיל `debug_mode` קבוע לכל משתמשי האתר החי. ב־GA4 היכנסו אל **ניהול → הצגת נתונים → DebugView** ובדקו את `whatsapp_clicked`.

האירוע צריך לכלול `source=accountant`, `medium=referral`, `campaign=partners-2026`, `content=personal`, `referrer_code=roynetanel` ו־`button_location=primary` או `secondary`.

ודאו שאינם מופיעים: `income`, `deposit`, `balance`, `recommended_amount`, `tax_benefit`, `phone`, `whatsapp_message`, `message_content` או תוכן ההודעה.

## הערות פרשנות

- `content` הוא `utm_content` שיווקי מותר; אין לבלבל אותו עם `message_content`, `content_text` או `whatsapp_message`, שנחסמים.
- מדדי הסשן המובנים של GA4 וממדי ה־Attribution הפנימיים עשויים להציג ערכים שונים, משום שהם משתמשים בכללי ייחוס שונים.
- `whatsapp_clicked` מודד לחיצה אמיתית על הכפתור בלבד, לא שליחה או קריאת הודעה.
